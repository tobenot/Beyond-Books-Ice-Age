测试指南

1. 测试类型
- 单元测试: 测试独立的函数和服务
- 集成测试: 测试多个服务之间的交互
- 流程测试: 测试完整的游戏流程
- 快照测试: 测试UI组件渲染

2. 测试文件组织
/tests
  /services      - 服务层单元测试
  /components    - UI组件测试
  /integration   - 集成测试
  /flows         - 流程测试
  setup.ts       - 测试环境配置

3. 运行测试
- npm test          运行所有测试
- npm run coverage  生成测试覆盖率报告
- npm test -- -t "测试名称" 运行特定测试

4. 编写测试建议
- 每个测试只测试一个功能点
- 使用有意义的测试名称
- 遵循 Arrange-Act-Assert 模式
- 避免测试之间的依赖

5. 常见测试场景
- 标签更新和读取
- 存档和读档
- 卡牌抽取和效果
- 地点切换
- 战斗系统
- 物品系统
